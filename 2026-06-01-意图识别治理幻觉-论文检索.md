# 2026-06-01 意图识别治理幻觉论文检索

关联笔记：[[硕士论文进展]]

## 本次检索结论

截至 2026-06-01，和“**使用意图识别/查询意图建模来治理幻觉**”最贴近的最新论文，主要集中在 2025 年到 2026 年初，且大多不直接叫“intent recognition for hallucination mitigation”，而是落在下面四条技术线上：

1. **意图幻觉评测**：证明模型会漏掉用户约束、误解用户约束。
2. **查询缺陷分类 / 查询检查点**：在回答前先判断输入是否需要验证。
3. **澄清式问答 / 交互式路由**：意图不确定时先追问，而不是直接答。
4. **医疗场景验证型 RAG**：在医疗 QA 中先验证证据，再生成回答。

你的论文和这条线的关系非常直接：你现在做的是**医疗 RAG 的上游治理**，核心是“多意图识别 + 输入质量评分 + 路由决策（直接答 / 改写 / 追问 / 保守回答）”。这正好可以被表述为：**将幻觉治理从输出端后处理，前移到输入端意图与语义质量控制**。

## 论文 1

### Beyond Facts: Evaluating Intent Hallucination in Large Language Models

- 作者：Yijie Hao, Haofei Yu, Jiaxuan You
- 时间：ACL 2025
- 链接：https://aclanthology.org/2025.acl-long.349/

#### 摘要

这篇论文提出了 **Intent Hallucination** 概念：LLM 面对带多重条件、复杂约束的查询时，不一定只是“事实错”，还会出现**漏掉用户意图的一部分**，或者**凭空补出用户没说过的约束**。作者构建了 **FAITHQA** 基准，共 20,068 个样本，覆盖 query-only 和 RAG 两种设定，并提出自动评测指标 **INTENT CONSTRAINT**。

#### 主要贡献

1. 把“幻觉”从事实层扩展到“意图对齐层”。
2. 提出 FAITHQA 这一专门检测 intent hallucination 的 benchmark。
3. 给出自动化指标 INTENT CONSTRAINT，用于检测遗漏/误解用户约束。

#### 和你论文的关系

这是你论文最应该引用的核心工作之一。它能直接支撑你的问题定义：**医疗问答中的风险不只是回答内容编造，还包括对患者问题意图的遗漏、混淆和误读**。你现在做多意图识别，本质上就是在减少这类 intent hallucination。

#### 你可以学习的思路

- 不要把“幻觉”只定义成事实错误，可以定义成：
  - `factual hallucination`
  - `intent hallucination`
  - `routing hallucination`：本该追问却直接答，本该保守却过度生成
- 你的 `Q_sem` 可以增加一个更明确的目标解释：它不是普通质量分，而是**输入端意图完整性与可答性风险分**。

#### 可借鉴的实验方法

- 额外构造一批**多约束患者问题**：
  - 同时包含症状、时长、既往史、用药史、期望建议
  - 检查模型是否漏识别某一类 intent
- 不只评分类准确率，还评“是否覆盖所有关键 intent”

#### 可借鉴的数据集/实验设计/指标

- 数据集：FAITHQA（通用域，但思路很适合迁移）
- 设计：
  - 单意图 vs 多意图
  - 清晰输入 vs 含缺失条件输入
  - query-only vs RAG
- 指标：
  - micro-F1 / macro-F1 之外，新增 `intent coverage`
  - `all-intents-hit rate`
  - `critical-intent-miss rate`

#### 可引用的地方

- 绪论：说明幻觉不仅是事实错误，也包含意图遗漏和误解。
- 相关工作：说明已有工作开始从“intent hallucination”视角重新定义问题。
- 方法动机：证明你的多意图识别不是附属模块，而是幻觉治理前置模块。

## 论文 2

### Lightweight Query Checkpoint: Classifying Faulty User Queries to Mitigate Hallucinations in Large Language Model Question Answering

- 作者：Minjoo Son, Jonghak Jang, Misuk Kim
- 时间：Findings of ACL 2025
- 链接：https://aclanthology.org/2025.findings-acl.756/

#### 摘要

论文关注一个和你非常接近的问题：不少幻觉不是回答器本身坏，而是**用户查询带有错误前提、上下文不足或语言歧义**。作者提出 **LQC（Lightweight Query Checkpoint）**，在回答前先用一个小型分类器判断该问题是否属于“需要验证”的查询，再决定是否进入回答流程。

#### 主要贡献

1. 明确定义了“需要验证”的 defective query 类型。
2. 构建 clear / defective 二分类数据。
3. 用较小模型中间层 hidden states 做前置判别，减少幻觉且保持回答质量。

#### 和你论文的关系

这篇论文几乎可以作为你“输入质量控制”模块的最近邻相关工作。你的 `Q_sem` 比它更细，不只是二分类，而是把输入拆成 `Q_intent / Q_text / Q_med / Q_route` 四部分。因此你可以把自己的贡献表述为：**从 binary checkpoint 扩展到 multi-factor semantic quality routing**。

#### 你可以学习的思路

- 把“要不要直接答”视为一个单独任务，而不是生成模型顺带完成。
- 先做 `verification-needed detection`，再做下游路由。
- 你完全可以把 `Q_sem` 的一个阈值版本实现成 LQC 式二分类基线。

#### 可借鉴的实验方法

- 构造三类坏输入：
  - 错误医学前提
  - 关键信息缺失
  - 歧义表达 / 多意图纠缠
- 做两个任务：
  - 任务 A：`need_verification` 二分类
  - 任务 B：四路路由 `direct / rewrite / ask / conservative`

#### 可借鉴的数据集/实验设计/指标

- 数据集：作者自建 clear / defective query 数据，并在多个 QA 数据集上测试
- 设计：
  - 无 checkpoint
  - binary checkpoint
  - 你的 `Q_sem` 四路路由
- 指标：
  - 分类：F1、AUC
  - QA：factuality、hallucination rate、answer quality
  - 路由：保守率、追问触发率、误杀率

#### 可引用的地方

- 相关工作：输入缺陷检测 / query verification
- 方法设计：说明你的路由思想并非拍脑袋，而是与最新 query checkpoint 范式一致

## 论文 3

### When in Doubt, Ask First: A Unified Retrieval Agent-Based System for Ambiguous and Unanswerable Question Answering

- 作者：Long S. T. Nguyen, Quynh T. N. Vo, Hung C. Luu, Tho T. Quan
- 时间：IJCNLP/AACL 2025 Findings
- 链接：https://aclanthology.org/2025.findings-ijcnlp.27/

#### 摘要

这篇论文明确讨论：高风险闭域 QA 中，**模糊问题和不可回答问题**是幻觉的重要来源。作者提出 **URASys**，在意图不确定时先进行轻量拆解与双阶段检索，并在需要时主动发起澄清提问；对不可回答情况则显式输出无法回答，以避免幻觉。

#### 主要贡献

1. 将标准问题、歧义问题、不可回答问题放进一个统一 QA 框架。
2. 在意图不确定时引入交互式 clarification。
3. 发布双语 ambiguous subset，用于评估澄清式 QA。

#### 和你论文的关系

这篇论文直接支持你当前的路由目标，尤其是“**追问/保守回答**”这一步。你当前进展里已经指出：低质量输入太少，所以 `Q_sem` 很少触发追问/保守回答。URASys 恰好告诉你：**必须专门构造 ambiguous / unanswerable 测试集，否则这个模块无法被严肃验证**。

#### 你可以学习的思路

- 把“追问”设计成可评测的系统行为，而不是 demo 行为。
- 把 `ask-first` 作为显式 policy。
- 将“不可回答”从失败案例变成正确路由结果。

#### 可借鉴的实验方法

- 构造专门的低质量医疗输入集：
  - 信息缺口型：年龄/性别/病程缺失
  - 术语歧义型：症状表达模糊
  - 多意图冲突型：问诊 + 用药 + 检查建议混在一起
  - 不可安全回答型：要求诊断或处方但证据不足
- 比较三种策略：
  - 直接 RAG
  - 改写后 RAG
  - 先追问 / 保守回答

#### 可借鉴的数据集/实验设计/指标

- 数据集：Vietnamese / English QA benchmarks + ambiguous subsets + academic advising tasks
- 设计：
  - ambiguous / standard / unanswerable 分桶评测
  - 人评 clarity / trustworthiness
- 指标：
  - factual accuracy
  - unanswerable handling accuracy
  - human-rated clarity
  - trustworthiness

#### 可引用的地方

- 相关工作：澄清式 QA、ambiguous question handling
- 实验设计：为你构造低质量输入测试集提供先例

## 论文 4

### VeReaFine: Iterative Verification Reasoning Refinement RAG for Hallucination-Resistant on Open-Ended Clinical QA

- 作者：Pakawat Phasook, Rapepong Pitijaroonpong, Jiramet Kinchagawat, Amrest Chinkamol, Tossaporn Saengja, Kiartnarin Udomlapsakul, Jitkapat Sawatphol, Piyalitt Ittichaiwong
- 时间：BioNLP 2025
- 链接：https://aclanthology.org/2025.bionlp-share.34/

#### 摘要

这是和你场景最接近的医疗论文之一。作者提出 **VeReaFine**，通过“检索 -> 验证 -> 生成/修正”的迭代闭环，确保回答中的每个关键陈述都有证据支撑。系统在证据不足时会生成 feedback query 继续检索，最终在 ClinIQLink 的 “LLM Lie-Detector” 任务上取得更强 factuality。

#### 主要贡献

1. 在医疗 open-ended QA 中显式加入 verifier。
2. 用 feedback query 弥补证据缺口。
3. 在临床共享任务上减少多跳错误，摘要显示 inverse step-identification error 降低 26%。

#### 和你论文的关系

它更偏**回答阶段治理**，你更偏**输入阶段治理**。这恰好能形成你论文的定位：VeReaFine 说明医疗 RAG 的确需要幻觉治理，但治理点可以更前移。你可以写成：

> 现有医疗幻觉治理多集中在检索后验证与回答修正；本文进一步探索输入端意图识别与语义质量路由，以在检索与生成前降低风险传播。

#### 你可以学习的思路

- 你的系统后续可和 verifier 组合成“两道闸”：
  - 闸 1：输入端 `Q_sem` 路由
  - 闸 2：回答端 evidence verification
- 论文答辩时，这种前后联合架构会更完整。

#### 可借鉴的实验方法

- 增加一组端到端对照实验：
  - 原始 RAG
  - `Q_sem` 路由 + RAG
  - `Q_sem` 路由 + RAG + verifier
- 即使你现在只做前半段，也可以把第三种作为 future work 原型。

#### 可借鉴的数据集/实验设计/指标

- 数据集：BioNLP 2025 ClinIQLink “LLM Lie-Detector”
- 设计：
  - evidence insufficient 时继续检索
  - 回答后逐条检查 claim support
- 指标：
  - factuality
  - reasoning quality
  - multi-hop error

#### 可引用的地方

- 绪论：医疗场景下 hallucination 风险更高
- 相关工作：医疗 RAG 验证式治理
- 讨论：你的方法属于“更前一层”的风险过滤

## 论文 5

### From Misleading Queries to Accurate Answers: A Three-Stage Fine-Tuning Method for LLMs

- 作者：Guocong Li, Weize Liu, Yihang Wu, Ping Wang, Shuaihan Huang, Hongxia Xu, Jian Wu
- 时间：Findings of ACL 2025
- 链接：https://aclanthology.org/2025.findings-acl.65/

#### 摘要

作者指出：LLM 对输入查询质量非常敏感，尤其是当问题本身含有**误导性或不准确前提**时。为此，他们提出三阶段微调：先识别误导信息，再纠正误导信息，最后基于纠正后的查询回答。论文在 hallucination detection 和 QA 任务上都得到更好的准确性和 factuality。

#### 主要贡献

1. 从“修输出”转向“先修输入”。
2. 提出 detect -> correct -> answer 的三阶段范式。
3. 构建带 misleading information 的数据集用于训练和测试。

#### 和你论文的关系

这篇论文和你的关系在于：你也在做“先处理输入，再决定如何回答”。不同点是，它偏向**纠错式 query correction**，你偏向**质量评估 + 路由**。两者可以并列成同类工作，并突出你的创新点是：

- 面向医疗场景
- 多意图而不是单纯 misleading premise
- 既支持改写，也支持追问/保守回答

#### 你可以学习的思路

- 把 `rewrite` 路由单独做成一个实验分支。
- 对“误导医学前提”构造专门测试集。
- 在论文中明确区分：
  - wrong premise
  - missing information
  - ambiguous intent

#### 可借鉴的实验方法

- 做四路 ablation：
  - 直接答
  - 仅改写
  - 仅追问
  - `Q_sem` 智能路由

#### 可借鉴的数据集/实验设计/指标

- 数据集：3 个 hallucination detection / QA 数据集 + 2 个自建 misleading query 数据集
- 指标：
  - 准确率
  - factuality
  - hallucination detection ability

#### 可引用的地方

- 相关工作：input correction / misleading query handling
- 方法动机：说明输入问题质量是 hallucination 的直接来源

## 论文 6

### How DDAIR you? Disambiguated Data Augmentation for Intent Recognition

- 作者：Galo Castillo-López, Alexis Lombard, Nasredine Semmar, Gaël de Chalendar
- 时间：EACL 2026
- 链接：https://aclanthology.org/2026.eacl-short.20/

#### 摘要

这篇论文不是直接做 hallucination，但对你当前“弱监督 + 多意图分类”非常关键。作者发现：用 LLM 做意图识别数据增强时，常会生成**语义上更接近其他类别**的模糊样本，导致训练集被污染。DDAIR 用 Sentence Transformers 检测这类歧义增强样本，并迭代重生成更干净的样本。

#### 主要贡献

1. 明确指出 LLM 数据增强会制造“意图歧义样本”。
2. 用 embedding 相似度识别错误增强样本。
3. 提出迭代重生成流程，改善低资源 intent recognition。

#### 和你论文的关系

这篇论文对你目前的训练流程非常有用，因为你现在已经在用：

- 人工种子集
- LLM 弱标签
- 质量筛选

你的下一步完全可以借鉴 DDAIR，把“**弱标签样本是否更像别的意图类**”纳入筛选规则。它能直接增强你当前 weakest link：**弱标签噪声控制**。

#### 你可以学习的思路

- 对每条 weak sample 计算：
  - 与目标标签 prototype 的相似度
  - 与次高标签 prototype 的间隔 margin
- 低 margin 样本降权或丢弃。
- 对多意图样本可改成“多正类中心距离 + 非目标类最大相似度”。

#### 可借鉴的实验方法

- 在你当前弱监督流程上加一组消融：
  - 无 embedding 过滤
  - 有 embedding ambiguity 过滤
  - 有过滤 + 人工抽检修正

#### 可借鉴的数据集/实验设计/指标

- 场景：低资源 intent recognition
- 方法关键词：LLM augmentation, ambiguity detection, iterative regeneration
- 指标：
  - micro-F1 / macro-F1
  - label ambiguity rate
  - filtered sample keep rate

#### 可引用的地方

- 相关工作：意图识别中的数据增强与歧义控制
- 方法部分：支持你当前弱标签筛选策略进一步升级
