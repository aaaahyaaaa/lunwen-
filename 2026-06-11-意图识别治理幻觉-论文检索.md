## 论文 1

### Uncertainty-Aware Clarification in LLM Agents with Information Gain

- 作者：Mohamad Jaber, Rishabh Iyer, Bhaskar Marthi, Nicholas Monath
- 时间：arXiv 2026-06-04
- 链接：[arXiv](https://arxiv.org/abs/2606.03135)

#### 摘要

这篇论文研究 LLM agent 什么时候应该先澄清、而不是直接行动或回答。核心思想是：当模型对用户目标存在不确定性时，不应只靠启发式规则决定是否追问，而应估计 **问一个问题之后能带来多少信息增益**，再决定是否发起 clarification。

#### 主要贡献

1. 把 clarification 触发条件从简单阈值推进到 **uncertainty-aware decision**。
2. 引入 **information gain** 视角，衡量“这次追问值不值得问”。
3. 说明 agent 的澄清行为可以做成显式决策模块，而不是附属 prompt 技巧。

#### 和你论文的关系

这篇论文非常适合强化你当前的 `Q_route` 思路。你现在已经在做：

- 意图识别
- `Q_sem`
- `direct / rewrite / ask / conservative` 路由

它能帮助你把 `ask` 写得更像一个正式研究问题：

> 当患者问题存在多意图纠缠、关键信息缺失或语义不确定时，系统应根据“不追问的风险”和“追问可能带来的信息收益”来决定是否进入 ask 路由。

#### 你可以学习的思路

- 在 `Q_route` 中加入“**不确定性**”概念，而不只是静态质量分。
- 把追问条件从“低于阈值”升级成“**预期收益是否大于交互成本**”。
- 设计一个更细的风险标签：
  - `worth-asking`
  - `answerable-with-rewrite`
  - `unsafe-even-after-clarification`

#### 可借鉴的实验方法

- 给你的低质量医疗输入集新增人工标注：
  - 这条样本是否需要追问
  - 若追问，是否能显著减少误答风险
  - 若不追问，最可能错在哪个槽位
- 对比三种策略：
  - 固定阈值 `Q_route`
  - 风险规则驱动 `Q_route`
  - 不确定性/信息收益驱动 `Q_route`

#### 可借鉴的数据集 / 实验设计 / 指标

- 论文主张可直接迁移，但具体任务细节需要正式读 PDF 再细抠。
- 你可直接新增的指标：
  - `necessary-ask recall`
  - `unnecessary-ask rate`
  - `ask information gain proxy`
  - `unsafe direct-answer rate`

#### 可引用的地方

- 方法章节：支撑 `ask` 是一个 uncertainty-aware policy。
- 讨论章节：解释为什么“该不该追问”不能只由固定规则决定。

## 论文 2

### SCICONVBENCH: Benchmarking LLMs in Scientific Conversational Question Asking

- 作者：Xuanang Lv, Chengjin Xu, Ruishan Liu, Binxuan Huang, Bangzheng Li, Wei Wang, Steven Bethard, Zhenqiang Wang, Xiangliang Zhang
- 时间：arXiv 2026-05-25
- 链接：[arXiv](https://arxiv.org/abs/2605.18896)

#### 摘要

这篇论文提出一个专门评测 LLM **在对话中提问质量** 的 benchmark。虽然场景是 scientific conversation，不是医疗 RAG，但它的重要价值在于：它把“模型能不能提出有效澄清问题”当成一个独立能力来评测，而不是只看最终答案好不好。

#### 主要贡献

1. 把 **question asking quality** 单独 benchmark 化。
2. 明确澄清问题的评估不只看流畅度，还要看是否命中信息缺口。
3. 为“追问质量”提供了比最终 QA 分数更细的分析入口。

#### 和你论文的关系

你的论文目前已经有：

- 多意图识别指标
- 弱监督质量指标
- `Q_sem` 相关指标

但如果后面要把 `ask` 作为真正的贡献点，你还缺一套“**追问本身的评估指标**”。这篇论文正好补这个缺口。

#### 你可以学习的思路

- 把实验拆成两个阶段：
  - 阶段 1：是否正确触发追问
  - 阶段 2：追问内容是否命中关键缺失医学信息
- 对你的医疗 ask 模块建立一个小型 benchmark，而不是只在最终回答上间接观察。

#### 可借鉴的实验方法

- 为每个低质量医疗问题人工补一个：
  - `critical missing slot`
  - `acceptable clarification question`
  - `bad clarification question`
- 然后评估系统提出的追问是否覆盖关键槽位。

#### 可借鉴的数据集 / 实验设计 / 指标

- 你可以新增的指标：
  - `critical-slot hit rate`
  - `clarification specificity`
  - `clarification usefulness`
  - `over-asking rate`

#### 可引用的地方

- 实验章节：支撑你把 ask 单独评测，而不只看最终回答。
- 未来工作：说明可进一步建设医疗澄清 benchmark。

## 论文 3

### Information Optimization Preference Optimization for Clarifying Question Generation in Large Language Models

- 作者：Laura Perez-Beltrachini, Sowmya Somanath, Marek Rei, Helen Hastie, Aydar Bulatov, Verena Rieser
- 时间：arXiv 2026-03-01
- 链接：[arXiv](https://arxiv.org/abs/2603.00656)

#### 摘要

这篇论文提出 **InfoPO**，把 clarifying question generation 的优化目标从“语言自然”推进到“**信息优化**”。也就是说，好的追问不只是礼貌、顺畅，还要最大化后续能获得的有效信息。

#### 主要贡献

1. 明确提出 clarifying question generation 的核心目标应是 **information optimization**。
2. 用 preference optimization 思路训练模型生成更有信息价值的澄清问题。
3. 强化了“少问但问准”的设计原则。

#### 和你论文的关系

你当前论文还没有正式展开 `ask` 生成模块，但这篇论文对你非常有用，因为它给了你一个很自然的扩展方向：

> 路由器先判断该不该追问；若需要追问，则优先提出能最大化补全关键医学槽位的信息型问题。

这和你现在想解决的“多意图 + 输入不完整 + 医疗 RAG 误答”高度一致。

#### 你可以学习的思路

- 把追问目标从“问得像不像人”改成“**问得能不能补关键医学信息**”。
- 约束追问优先覆盖：
  - 症状持续时间
  - 年龄/性别
  - 部位/程度
  - 已有检查与用药

#### 可借鉴的实验方法

- 让标注者给每条低质量医疗输入补一个“最优下一问”。
- 比较：
  - 通用 LLM 追问
  - 模板式追问
  - 信息优化导向追问

#### 可借鉴的数据集 / 实验设计 / 指标

- 你可新增的指标：
  - `one-turn clarification success`
  - `critical-info gain`
  - `question efficiency`
  - `follow-up answer safety`

#### 可引用的地方

- 相关工作：说明 clarifying question generation 不应只追求语言自然度。
- 讨论章节：支撑“少问但问准”的交互原则。

### Med-HEAL: A Dynamic Benchmark for Evaluating Hallucinations in LLMs for Medical Applications

这篇论文并不直接研究 intent recognition，但它非常适合作为你医疗实验的 **结果评测参考**。论文构建了一个面向医疗应用的动态 hallucination benchmark，强调医疗领域需要比通用问答更严格的错误评估。

#### 主要贡献

1. 提供医疗 hallucination 的专门 benchmark 视角。
2. 强调不同类型医疗错误的风险不对等。
3. 适合为你的“路由是否真的降低高风险误答”提供外部评测逻辑。

#### 和你论文的关系

你的核心创新在输入侧，但最终还是要回答一个问题：

> 输入端的意图识别与路由治理，是否真的减少了医疗 hallucination？

这篇论文可以帮助你把实验闭环补完整：

- 前端：识别多意图、信息缺失、医学相关性
- 中端：选择 `direct / rewrite / ask / conservative`
- 后端：评测最终医疗回答中的 hallucination 风险

#### 你可以学习的思路

- 为最终回答增加风险分层：
  - 低风险泛化错误
  - 中风险误导性建议

#### 可借鉴的实验方法

- 在你自己的低质量医疗测试集上，对最终输出人工标记：
  - 是否存在 hallucination
  - 风险等级
  - 是否本可通过追问避免
- 对比：
  - 直接 RAG
  - `Q_sem` 路由
  - `Q_sem` 路由 + ask/conservative

#### 可借鉴的数据集 / 实验设计 / 指标

- 你可新增的指标：
  - `medical hallucination rate`
  - `high-risk hallucination rate`
  - `avoidable hallucination rate`
  - `route-adjusted safety gain`

#### 可引用的地方

- 绪论：说明医疗 hallucination 的评测不能只用通用 factuality 指标。
- 实验章节：支撑你引入风险分层评估。

## 对当前硕士论文最直接的启发

结合今天补录的论文，我认为你下一步最值得立刻落地的不是再扩标签，而是把 `Q_route` 和低质量输入实验再往前推进半步：

### 1. 把 `Q_route` 写成策略门控器，而不只是质量子分数

你现在可以把方法描述从：

> `Q_route` 衡量问题是否适合直接进入 RAG

改成：

> `Q_route` 是一个回答前策略门控信号，用于判断当前输入应进入 `direct`、`rewrite`、`ask` 或 `conservative` 路由，以减少信息缺失、意图未明和医学风险带来的 hallucination。

### 2. 给低质量输入集补三类标注

建议你新增 50 到 100 条小规模人工集，先做出一个像样的 proof-of-concept：

- `need_clarification`: 是否必须追问
- `critical_missing_slot`: 最关键缺失信息是什么
- `unsafe_if_direct`: 如果直接回答，是否高风险

这 3 个标注会让你后面的实验一下子更像论文，而不是工程流水线。

### 3. 把 ask 的评估独立出来

建议不要只报“最终回答正确率”，再补一组 ask 专项指标：

- `necessary-ask recall`
- `unnecessary-ask rate`
- `critical-slot hit rate`
- `recognized-but-answered rate`

其中 `recognized-but-answered rate` 特别值得保留，因为它和你前两天记录的 under-specification 论文能形成很强呼应。

### 4. 最值得写进论文的实验对比

你后面可以优先做这 4 个系统对比：

1. `Direct RAG`
2. `All low-score -> rewrite`
3. `Q_sem route`
4. `Q_sem route + ask/conservative`

如果精力允许，再补一个：

5. `Q_sem route + uncertainty-aware ask trigger`

即使第 5 个先只做规则近似版，也已经足够形成方法升级。
