## 论文 1

### INSCIT: Information-Seeking Conversations with Mixed-Initiative Interactions

- 作者：Zeqiu Wu, Ryu Parish, Hao Cheng, Sewon Min, Prithviraj Ammanabrolu, Mari Ostendorf, Hannaneh Hajishirzi
- 首次公开时间：2022-07-02
- 链接：[arXiv](https://arxiv.org/abs/2207.00746)

#### 摘要

论文直接研究信息寻求对话中的核心问题：用户问题可能是 **under-specified**、也可能暂时 **不可回答**。作者提出 InSCIT 数据集，把系统动作显式分成三类：**直接回答**、**追问澄清**、**提供相关信息**。这比传统 QA 更接近真实系统，因为系统不再被迫每次都立即给答案。

#### 主要贡献

1. 把信息寻求对话建模成 **mixed-initiative interaction**，而不是单一路径的“有问必答”。
2. 构造了 805 组人机对话、4.7K user-agent turns 的数据集，显式包含 `answer / ask / provide relevant info` 三种系统动作。
3. 同时提供 evidence passage identification 和 response generation 两个子任务，说明“先检索再决定怎么回应”是一个完整问题。

#### 和你论文的关系

这篇论文和你的研究关系最直接。它相当于给你的 `Q_route` 提供了一个早期形式化原型：

- `direct RAG` 对应直接回答。
- `ask` 对应 clarifying question。
- `conservative` 可以映射到“先提供相关信息但不下结论”。

你的论文可以进一步把这个思想收缩到医疗场景：当患者输入缺少关键槽位时，不应直接检索并生成医学建议，而应先触发追问或保守回答。

#### 你可以学习的思路、实验方法、数据集、实验设计、评测指标、可引用点

- 思路：把“先答还是先问”设为显式分类/路由任务。
- 实验方法：比较 `always-answer` 与 `route-then-answer` 两条 pipeline。
- 数据集：可把 InSCIT 的动作标签思想迁移到你的医疗问题数据上，标注 `answer / ask / rewrite / conservative`。
- 实验设计：专门构造 `信息充分`、`信息不足`、`无法安全回答` 三类子集。
- 评测指标：`route accuracy`、`ask recall`、`unsafe direct answer rate`、`post-route retrieval gain`。
- 可引用点：真实信息寻求系统必须允许系统主动发起澄清，而不是把每个输入都当成可直接回答的问题。

## 论文 2

### Recognizing Medical Search Query Intent by Few-shot Learning

- 作者：Yaqing Wang, Song Wang, Yanyan Li, Dejing Dou
- 首次可检索时间：2022-07（SIGIR 2022 proceedings）
- 链接：[SIGIR 2022 Proceedings](https://sigir.org/sigir2022/program/proceedings/)

#### 摘要

论文聚焦**医学搜索查询意图识别**。作者指出医学查询通常很短、噪声高、缺乏严格句法结构，还需要医学背景知识才能正确理解；同时很多细粒度意图只有很少标注样本。为此，作者提出 few-shot 方法 `MEDIC`，结合搜索日志中的 co-click queries 做弱监督，并融合医学知识图谱、句法知识和通用语言模型表征。

#### 主要贡献

1. 直接面向**医学查询意图识别**，而不是泛领域 intent classification。
2. 在少样本条件下，引入 **co-click weak supervision**，缓解医学标签稀缺问题。
3. 把 **医学知识图谱 + 句法信息 + 通用语义表示** 融合到 query encoder 中。

#### 和你论文的关系

这篇论文和你当前的 `12 类医学多意图识别` 最贴近。它能支持你论文里的一个关键论点：医疗输入治理首先不是生成，而是**理解患者问题背后的意图结构**。尤其是：

- 医疗查询短、乱、口语化，和你现在处理的患者问题非常相似。
- 少样本、弱监督、外部知识融合，都和你当前 `gold + weak labels + 规则/Q_sem` 的路线兼容。

它的不足是更偏单查询意图识别，还没有扩展到 `ask / rewrite / conservative` 路由；这正好给你留出创新空间。

#### 你可以学习的思路、实验方法、数据集、实验设计、评测指标、可引用点

- 思路：把医学输入理解建立在“少样本 + 弱监督 + 领域知识”三者结合上。
- 实验方法：除 TF-IDF + LR 外，可增加一个轻量 BERT/Chinese medical encoder 对比，并测试是否加入词法/实体特征。
- 数据集：你可继续用 MedDialog-CN 抽样问题，同时补充医学术语词典、症状词典、药物词典作为知识特征。
- 实验设计：做 `仅文本`、`文本+医学词典`、`文本+知识特征`、`文本+弱监督` 的消融。
- 评测指标：`micro-F1`、`macro-F1`、`few-shot intent recall`、`rare intent F1`。
- 可引用点：医学搜索意图识别具有短文本、噪声高、标签稀缺和领域知识依赖强的典型特点。

## 论文 3

### Generating Clarifying Questions with Web Search Results

- 作者：Ziliang Zhao, Zhicheng Dou, Jiaxin Mao, Ji-Rong Wen
- 首次可检索时间：2022-07（SIGIR 2022 proceedings）
- 链接：[SIGIR 2022 Proceedings](https://sigir.org/sigir2022/program/proceedings/)

#### 摘要

论文研究如何为搜索系统生成高质量澄清问题。作者指出，已有方法过度依赖静态知识库，遇到缺少明确实体或缺少结构化描述的查询时，生成的问题往往不够有信息量。为此，作者直接利用**初始搜索结果**中的上下文，抽取和排序描述候选，再用排序模型和生成模型提升澄清问题质量。

#### 主要贡献

1. 提出 **retrieval-backed clarification** 思路，用搜索结果反过来帮助问出更好的问题。
2. 说明澄清问题生成不应只基于 query 表面文本，还应利用初始检索证据。
3. 给出规则抽取、learning-to-rank 和生成式方法结合的工程路线。

#### 和你论文的关系

这篇论文对你的 `ask` 路由非常有用。你的系统以后不必只是机械追问“请描述更清楚”，而可以：

1. 先做一次轻量检索。
2. 看候选证据中分歧最大的缺口是什么。
3. 再问一个**最能减少检索不确定性**的问题。

在医疗场景里，这能把澄清问题从泛泛追问变成更有针对性的槽位补全，例如“疼痛部位”“持续时间”“年龄/孕产状态”“是否已有检查结果”。

#### 你可以学习的思路、实验方法、数据集、实验设计、评测指标、可引用点

- 思路：先检索，再追问；追问服务于后续证据收敛。
- 实验方法：比较 `generic ask`、`template ask`、`retrieval-backed ask`。
- 数据集：从你的患者问题中构造缺失槽位样本，再配套人工写澄清问题。
- 实验设计：看追问后是否提升检索一致性和最终答案安全性，而不是只看追问句子的流畅度。
- 评测指标：`clarification usefulness`、`post-clarification Recall@k`、`slot recovery rate`、`unsafe answer reduction`。
- 可引用点：澄清问题的价值在于减少意图歧义和检索不确定性，而非仅生成自然语言问题。

## 论文 4

### Curriculum Contrastive Context Denoising for Few-shot Conversational Dense Retrieval

- 作者：Kelong Mao, Zhicheng Dou, Hongjin Qian
- 首次可检索时间：2022-07（SIGIR 2022 proceedings）
- 链接：[SIGIR 2022 Proceedings](https://sigir.org/sigir2022/program/proceedings/)

#### 摘要

论文关注对话检索中的一个非常实际的问题：**不是所有历史轮次都对当前查询有用**。作者指出，冗余历史会干扰当前查询意图理解，因此提出 `COTED`，用 curriculum learning 和 contrastive learning 逐步学习从带噪历史中恢复对当前问题真正有帮助的上下文表示，并在 CAsT-19 / CAsT-20 上验证有效性。

#### 主要贡献

1. 把 conversational retrieval 中的关键问题从“如何拼接更多上下文”改成“如何去掉无关上下文”。
2. 用对比学习构造 `noised sample` 与 `denoised sample`，强化上下文去噪能力。
3. 在 few-shot conversational search 条件下证明，合理去噪比简单累加历史更有效。

#### 和你论文的关系

这篇论文非常适合映射到你的 `Q_text`。医疗问答里也有类似问题：

- 患者会夹带情绪、背景叙述、重复话语。
- 多轮对话里并不是所有历史都该进入检索。
- 有些输入应该先抽取关键槽位或做轻量改写，而不是把原话整段扔进 RAG。

因此你可以把 `rewrite` 再拆成两个更细的动作：

- `denoise / normalize`：保留关键信息，去掉噪声。
- `clarify / ask`：缺关键信息时再追问。

#### 你可以学习的思路、实验方法、数据集、实验设计、评测指标、可引用点

- 思路：先判断哪些内容该保留，哪些上下文该丢弃。
- 实验方法：比较 `raw input`、`history concat`、`slot-only denoise`、`rewrite after denoise`。
- 数据集：可从 MedDialog-CN 多轮对话中抽取含冗余上下文的样本，人工标关键医疗槽位。
- 实验设计：分别测去噪对意图识别、检索、路由决策的影响。
- 评测指标：`retrieval Recall@k`、`route accuracy`、`noise removal precision`、`downstream F1 gain`。
- 可引用点：面向对话检索，更多上下文并不总是更好，输入去噪本身就是关键治理步骤。

## 与硕士论文进展的关系

结合 [`/Users/hanye/Documents/Obsidian Vault/硕士论文进展.md`](/Users/hanye/Documents/Obsidian%20Vault/%E7%A1%95%E5%A3%AB%E8%AE%BA%E6%96%87%E8%BF%9B%E5%B1%95.md)，这一桶最值得吸收的不是某个单一模型，而是一个更清晰的**输入治理分层框架**：

1. **先识别医学意图**：对应 `Recognizing Medical Search Query Intent by Few-shot Learning`。
2. **再判断信息是否足够**：对应 `INSCIT` 的 mixed-initiative 路由思想。
3. **若信息不足，决定是 ask 还是 rewrite**：
   - 缺关键槽位，用 `ask`。
   - 只是上下文省略或噪声高，用 `denoise/rewrite`。
4. **追问应服务检索**：对应 `Generating Clarifying Questions with Web Search Results`。
5. **改写前先去噪**：对应 `COTED`。

这意味着你可以把当前的 `Q_sem` 写得更像一个**前置治理控制器**：

```text
输入问题
-> 意图识别 / 多意图检测
-> 信息充分性判断
-> 路由决策：direct RAG / denoise-rewrite / ask / conservative
-> 再进入检索与生成
```

这比“直接做医疗 RAG”更容易形成论文创新点，因为你解决的是 **RAG 上游输入质量控制**。

## 我可以直接学习和复用的内容

### 1. 可提炼成论文方法的思路

- 把 `ask / rewrite / conservative` 从工程 heuristic 提升为显式路由任务。
- 把“是否可直接回答”做成回答前判别模块，而不是生成后再修补。

- 做组件消融：去掉意图特征、去掉医学词典、去掉澄清问题模块、去掉 conservative 路由。

- 你的创新可以定位为：**面向医疗 RAG 的输入侧意图识别与路由治理框架**，目标是在生成前降低 hallucination 风险。

## 本桶结论

`2022 Q3` 这一桶给你的启发，不是“已经有人做了医疗 RAG 幻觉治理”，而是更关键的一点：**在 RAG 时代之前，已经有一批工作把 ask、intent、clarify、denoise、mixed-initiative 这些上游模块分别做出来了。**

你的论文正好可以把这些分散模块整合成一个医疗场景下的统一框架：

- 先识别意图。
- 再判断输入是否适合直接检索。
- 不适合时选择 `ask / rewrite / conservative`。
- 最后才进入医疗 RAG。

这条线的创新性在于：**把 hallucination 治理前移到输入控制与路由决策，而不是只在回答生成后做纠偏。**
