## 论文汇总

### 1. Multifaceted Improvements for Conversational Open-Domain Question Answering

- 作者：Tingting Liang, Yixuan Jiang, Congying Xia, Ziqiang Zhao, Yuyu Yin, Philip S. Yu
- 首次可检索时间：2022-04-01
- 链接：[arXiv](https://arxiv.org/abs/2204.00266)
- 摘要：
  - 论文关注 open-domain conversational QA 中的三个前置问题：问题理解不准、passage 粗排过粗、训练时过度依赖 golden passage。
  - 作者提出 `MICQA`，把 question understanding、post-ranking 和 curriculum learning 联合起来，减少“问题没说清就直接答”的误差放大。
  - 在 `OR-QuAC` 上，作者报告相对既有 conversational OpenQA baseline 有稳定优势。
- 主要贡献：
  - 把“回答前的问题理解质量”明确建模为独立瓶颈。
  - 说明 query 解释方式会同时影响 retrieval 和 answer reading。
  - 给出一个很接近你 `direct RAG` 前置质量控制的两阶段框架：先把问题解释好，再决定检索和阅读。

### 2. Conversational Question Answering on Heterogeneous Sources

- 作者：Philipp Christmann, Rishiraj Saha Roy, Gerhard Weikum
- 首次可检索时间：2022-04-25
- 链接：[arXiv](https://arxiv.org/abs/2204.11677)
- 摘要：
  - 论文指出 follow-up question 往往把关键信息留在上下文里，直接拿短 query 去检索很容易错。
  - 作者提出 `CONVINSE`，先把当前问题和对话上下文转成 intent-explicit 的 structured representation，再统一去检索 `KB + text + table` 多源证据，最后用生成式模块融合回答。
  - 同时构建并发布 `ConvMix` 数据集，覆盖 `3000` 组真实用户对话、约 `16000` 个问题。
- 主要贡献：
  - 用显式结构化表示替代单纯 question rewriting，是“回答前判别/解释”的强版本。
  - 提供 `ConvMix` 这种更贴近真实多轮信息缺失场景的数据集。
  - 论文实验直接比较了 structured representation 与 question rewriting / question resolution，说明“只改写成完整句”未必最优。

### 3. Can Question Rewriting Help Conversational Question Answering?

- 作者：Etsuko Ishii, Yan Xu, Samuel Cahyawijaya, Bryan Wilie
- 首次可检索时间：2022-05
- 链接：[ACL Anthology](https://aclanthology.org/2022.insights-1.13/)
- 摘要：
  - 论文专门检验一个关键假设：`question rewriting` 真的能改善 conversational QA 吗？
  - 作者使用强化学习把 QR 和 QA 联训，并避免依赖每个目标 CQA 数据集都有重写标注。
  - 结果显示该方法基本只做到与 end-to-end baseline 持平，未能显著超越；作者进一步分析了 QR 难以转化为下游收益的原因。
- 主要贡献：
  - 这是一个很重要的“负结果”：`rewrite` 不是天然有效，必须看任务和评价目标。
  - 实验覆盖 `CoQA`、`QuAC`，并使用 `CANARD`、`QReCC` 等 QR 数据。
  - 评测指标包含 `F1`，在 `QuAC` 上还看 `HEQ-Q`、`HEQ-D`，适合作为你后面做 route ablation 的评测参考。

### 4. Pseudo Ambiguous and Clarifying Questions Based on Sentence Structures Toward Clarifying Question Answering System

- 作者：Yuya Nakano, Seiya Kawano, Koichiro Yoshino, Katsuhito Sudoh, Satoshi Nakamura
- 首次可检索时间：2022-05
- 链接：[ACL Anthology](https://aclanthology.org/2022.dialdoc-1.4/)
- 摘要：
  - 论文直接面向 clarifying QA：当用户问题缺少必要信息时，系统不应立刻回答，而应先追问以消歧。
  - 作者提出两件事：一是基于句法结构，把现有 QA 数据改造成 pseudo ambiguous question；二是基于 case frame 和 sentence structure 生成 clarifying question。
  - 实验显示，构造出来的伪歧义问题确实增加了难度，而引入澄清问题后可以恢复一部分 QA 性能损失。
- 主要贡献：
  - 给“低质量输入集怎么造”提供了很实用的方法。
  - 证明 `ask` 分支可以作为恢复性能的交互机制，而不只是兜底策略。
  - 这篇和你的 `under-specification handling` 方向最直接同构。

### 5. Conversational Search with Mixed-Initiative - Asking Good Clarification Questions backed-up by Passage Retrieval

- 作者：Yosi Mass, Doron Cohen, Asaf Yehudai, David Konopnicki
- 首次可检索时间：2022-05
- 链接：[ACL Anthology](https://aclanthology.org/2022.dialdoc-1.7/)
- 摘要：
  - 论文研究的不是“要不要澄清”，而是“澄清时问哪个问题最好”。
  - 作者把可能的用户意图近似为从文档库中检索出的 passages，再用双 BERT ranker 对候选 clarifying question 做重排。
  - 实验覆盖开放域 `ClariQ` 和一个 task-oriented customer support 数据集，并使用 `Recall@30` 作为主指标。
- 主要贡献：
  - 把 clarification question selection 明确转化为 retrieval + ranking 问题。
  - 说明“追问内容”也可以像检索一样被建模和评测。
  - 对你后面做医疗 ask 模板排序很有借鉴价值。

### 对你当前三/四路路由的映射

- `direct RAG`
  - 不是“医学相关就直接检索”，而是要先判断问题是否已经足够完整、焦点是否明确、是否能稳定命中证据。
- `rewrite`
  - 可以做，但需要像 `Can Question Rewriting Help CQA?` 那样证明 rewrite 后真的提升了下游检索或回答，而不是只让句子更通顺。
- `ask`
  - `Pseudo Ambiguous...` 和 `Mixed-Initiative...` 共同说明：ask 可以分成两个子问题，先判断是否需要追问，再从候选追问中选最有区分力的一句。
- `conservative`

## 我可以学习的思路

### 方法思路

- 把 hallucination 防治前移为 `intent-explicit understanding`，先做问题解释，再做检索和作答。
- 把 `rewrite` 与 `clarify` 拆成两条不同策略，而不是把两者混成“输入清洗”。
- 允许 `rewrite` 失败，甚至默认它不一定优于端到端模型；真正重要的是它是否提升下游。
- 用显式结构化表示替代纯文本改写，是值得你在医疗场景里尝试的方向。

### 可借鉴的实验方法

- 做 route-level ablation：
  - raw query 直接检索
  - rewrite 后检索
  - ask 后补全再检索
  - conservative 不检索
- 做“是否需要追问”的二分类，再做“该问哪一句”的排序任务，不要一上来直接端到端生成澄清问题。
- 针对 rewrite 分支，必须同时看文本质量指标和下游 QA/RAG 指标。
- 可以先构造 pseudo ambiguous 医疗问题，再验证 ask/rewrite 是否真的恢复回答质量。

### 可直接关注的数据集/设定

- `OR-QuAC`
  - 适合参考 conversational retrieval + reading 的两阶段评测方式。
- `ConvMix`
  - 适合学习如何把上下文隐含意图转成结构化表示，并在多源证据下评估。
- `CoQA` / `QuAC`
  - 适合参考 rewrite 分支的标准 QA 指标与负结果分析。
- `CANARD` / `QReCC`
  - 适合借鉴 conversational question rewriting 数据组织方式。
- `ClariQ`
  - 很适合拿来类比你的“是否追问 + 追问排序”子任务。
- 你自己的 `MedDialog-CN` 患者问题子集
  - 很适合仿照 `Pseudo Ambiguous...` 去做缺槽、缺部位、缺时长、缺对象、含多意图缠绕的伪歧义版本。

### 可模仿的实验设计

- `E1`：多意图识别。
- `E2`：输入完整性/歧义性判别。
- `E3`：route 决策（direct / rewrite / ask / conservative）。
- `E4`：ask 后或 rewrite 后，最终检索质量和回答可靠性是否提升。
- `E5`：对 `rewrite` 做负结果检验，验证它是否只是“写得更像标准问句”，但并未改善医疗检索。

### 可借鉴的评测指标

- 多标签分类：
  - `micro-F1`
  - `macro-F1`
  - `hamming loss`
- rewrite / QA：
  - `F1`
  - `HEQ-Q`
  - `HEQ-D`
  - 下游检索命中率或答案正确率变化
- clarification ranking：
  - `Recall@30`
- heterogeneous evidence QA：
  - `answer presence`
  - `P@1`

### 可引用点

- 在信息不完整的多轮问答中，先把隐含意图显式化，比直接把原始短问题送入 QA 更可靠。
- question rewriting 并非天然有效，必须通过下游问答收益验证。
- 澄清式交互可以被分解为“是否追问”与“追问哪一句”两个可独立评测的任务。
- 结构化意图表示比纯文本改写更有机会保留检索所需的关键槽位和关系约束。
