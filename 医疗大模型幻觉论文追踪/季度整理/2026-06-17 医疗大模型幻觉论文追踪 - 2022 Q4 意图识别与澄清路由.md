### 1. Conco-ERNIE: Complex User Intent Detect Model for Smart Healthcare Cognitive Bot

- 作者：Bolin Zhang, Zhiying Tu, Shaoshi Hang, Dianhui Chu, Xiaofei Xu
- 首次可检索时间：2022-12-08
- 链接：[DOI](https://doi.org/10.1145/3574135)
- 摘要：
  - 论文面向智能医疗问答机器人中的复杂用户意图识别，强调患者话语不仅有 `single intent`，还会出现 `multi-intent` 和 `implicit intent`。
  - 作者构建了 `CMedQ` 数据集，约 `16.8k` 条中文医疗 query，覆盖单意图、多意图和隐式意图，并同步构建了 `CMedKG` 医疗知识图谱，约 `45k` 实体、`283k` 关系。
  - 方法上提出 `Conco-ERNIE`，用概念共现模式增强 `ERNIE` 表征，再用注意力机制聚合语义特征与概念特征，以提升对显式与隐式意图的识别能力。
- 主要贡献：
  - 直接把“复杂医疗用户输入”建模为多意图/隐式意图问题，而不是普通单标签分类。
  - 给你当前的“患者问题多意图识别”提供了很贴近的任务设定与数据构造思路。
  - 说明上游 query 理解本身就是医疗问答可靠性的关键前提。

### 2. Medical QA Oriented Multi-Task Learning Model for Question Intent Classification and Named Entity Recognition

- 作者：Turdi Tohti, Mamatjan Abdurxit, Askar Hamdulla
- 首次可检索时间：2022-12-14
- 链接：[DOI](https://doi.org/10.3390/info13120581)
- 摘要：
  - 论文指出，医疗问句的 `intent classification` 和 `named entity recognition` 是问答系统自然语言理解模块的两个核心子任务，但现有工作常把它们割裂处理。
  - 作者提出基于 `ALBERT-BiLSTM` 的多任务学习模型，通过 encoder 参数共享，让底层网络同时吸收意图分类与实体识别特征。
  - 在中文在线医疗问题数据上，所提多任务模型在 precision、recall、F1 上优于基线，并比单任务模型有更好的泛化能力。
- 主要贡献：
  - 把“先识别问什么”和“先抽取问到的医学对象”合并建模。
  - 这对你未来做 `Q_intent` 和 `Q_med` 很有帮助，因为它暗示二者不应完全独立打分。
  - 如果你的 router 以后要决定 `direct / rewrite / ask`，意图和实体是否同步清晰，应该是一个联合信号。

### 3. A medical Q&A system with entity linking and intent recognition

- 作者：Fengming Guan, Taro Tezuka
- 首次可检索时间：2022-12-04
- 链接：[DOI](https://doi.org/10.1109/SSCI51031.2022.10022093)
- 摘要：
  - 论文提出一个基于医疗知识图谱的问答系统，核心流程是：先做 `intent recognition`、`named entity recognition`、`entity linking`，再用规则模板和 `Cypher` 到知识图谱中检索答案。
  - 系统专门处理用户问题中的非标准实体表达，把患者输入映射到知识图谱中的规范疾病名，再进入检索。
  - 实验中，意图识别用 `BERT-TextCNN`，在 `CMID` 数据集上准确率 `0.67`；NER 用 `BiLSTM-CRF`，在 `cMedQANER` 上准确率 `0.96`；实体链接用 `ESIM`，在 `Yidu-N7K` 上准确率 `0.96`。在 `200` 个问题的系统实验里，当用户不使用标准实体名或不使用模板特征词时，该系统明显优于只依赖模板匹配的方案。
- 主要贡献：
  - 这是很典型的“回答前先规范化输入”的系统论文。
  - 它证明了 query 中的非标准表达、别名、俗称，本质上就是会把后续检索/回答推向错误路径的上游噪声。
  - 对你当前的医疗 RAG 场景来说，这几乎就是 `rewrite / normalize / link` 路由的早期知识图谱版本。

### 4. Intent-aware Prompt Learning for Medical Question Summarization

- 作者：Leilei Zhang, Junfei Liu
- 首次可检索时间：2022-12-06
- 链接：[DOI](https://doi.org/10.1109/BIBM55620.2022.9995317)
- 摘要：
  - 论文聚焦 `consumer health question summarization`，目标是把患者原始问题压缩为“足以找到正确答案的最小必要问题”，从而提升自动医疗问答系统检索相关答案的成功率。
  - 作者认为，不同健康问题面向不同医学实体和不同提问目的，如果对所有样本使用同一套 prompt，会导致摘要无意义或质量不稳。
  - 因此提出 `InPL`，在统一 transformer 中串联 `question intent recognition module`、`dynamic prompt module` 和 `medical summarization generation module`，让摘要过程显式感知用户意图；实验表明该方法能生成更高质量、也更“factually correct”的摘要。
- 主要贡献：
  - 这是和你 `rewrite` 分支最贴近的一篇。
  - 它不是简单改写，而是“意图驱动的医学问题压缩与聚焦”。
  - 论文把 rewrite 的价值直接绑定到下游答案检索成功率与 factual correctness，而不是只看表面文本质量。

### 5. Intent Disambiguation for Task-oriented Dialogue Systems

- 作者：Andrea Alfieri, Ralf Wolter, Seyyed Hadi Hashemi
- 首次可检索时间：2022-10-16
- 链接：[DOI](https://doi.org/10.1145/3511808.3557516)
- 摘要：
  - 论文虽然不是医疗领域，但它解决的问题和你想做的 `ask` 路由非常同构：用户常无法在单轮话语里完整表达需求，导致意图分类出错。
  - 作者提出两步式澄清方案：
    - 先识别当前输入是否含糊、是否应该追问。
    - 再给出 top-N 候选意图，并把它们组织成 clarification question。
  - 实验显示，该方法在 `intent disambiguation accuracy` 和 `MRR` 上显著优于基于 popularity 的方法和原有意图排序模型。
- 主要贡献：
  - 这篇论文直接给出“回答前先判别是否需要澄清”的建模框架。
  - 它能帮助你把 `ask` 从规则触发升级为显式子任务：`need_ask` + `ask_target / candidate intents`。
  - 虽然不是医疗论文，但对于你想做的 `ask / rewrite / direct` 路由非常关键。

- 把 hallucination 治理前移到 `回答前动作选择`，而不是只在回答后做事实核验。
- 把 `rewrite` 定义成医学问题理解，而不是通用文本润色。

### 可引用点

- 患者 query 的风险不只来自“错别字”或“口语化”，更来自 `multi-intent`、`implicit intent` 和 `non-standard entity mention`。
- 医疗问答前的 rewrite，应该以“保留医学意图并提升下游 grounding 成功率”为目标，而不是单纯生成更短的问题。
- 当系统能够判断“当前输入不应直接答、而应先澄清”时，本质上就是在做 hallucination 的上游治理。
- 意图识别、实体识别和实体链接如果割裂建模，会放大错误传播；联合建模或级联约束更适合医疗问答场景。

## 本次结论

- `2022 Q4` 还没有成熟的“医疗 RAG 幻觉治理”论文，但已经出现了非常关键的上游模块共识：
  - 先识别复杂意图。
  - 先规范化医学实体。
  - 先做意图驱动的问题摘要或改写。
  - 对歧义输入先追问，再分类或检索。
- 这些工作和你的硕士课题不是松散相关，而是正好拼成你要做的医疗 RAG 前端路由器。
- 如果你把这些模块统一到一个 `input quality control + route decision` 框架里，你的论文就会比单纯做“多标签分类”更完整，也更接近 hallucination 治理这个主线。
