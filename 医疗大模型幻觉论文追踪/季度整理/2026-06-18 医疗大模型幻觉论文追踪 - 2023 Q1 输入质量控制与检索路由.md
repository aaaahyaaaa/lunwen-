# 医疗大模型幻觉论文追踪 - 2023 Q1 输入质量控制与检索路由

本次检索时间桶：2023 Q1（2023-01-01 到 2023-03-31）

检索说明：
- 本次只纳入在 `2023-01-01` 到 `2023-03-31` 之间首次发表、公开或可检索到的论文。
- 主题限定为：通过意图识别、澄清式交互、查询路由、回答前判别、query rewriting、intent clarification、under-specification handling 等方式治理 hallucination。
- 这一时间桶内“直接面向医疗 RAG + 先澄清/先改写/先路由”的论文很少，因此本次结果分为两类：
  - 直接相关：医疗 RAG 或显式 query routing / rewriting。
  - 可迁移相关：虽非医疗场景，但方法可以直接迁移到你的“医疗输入质量控制”主线。

## 一、论文汇总

### 1. Toolformer: Language Models Can Teach Themselves to Use Tools
- 作者：Timo Schick, Jane Dwivedi-Yu, Roberto Dessi, Roberta Raileanu, Maria Lomeli, Luke Zettlemoyer, Nicola Cancedda, Thomas Scialom
- 首次公开：2023-02-09
- 链接：[arXiv:2302.04761](https://arxiv.org/abs/2302.04761)
- 摘要：
  - 论文提出 `Toolformer`，让语言模型自行学习“何时调用工具、调用什么工具、如何把工具返回结果并入后续生成”。
  - 它把“先判断是否该直接回答”显式化，模型不再一味依赖参数记忆，而是可选择调用搜索、QA、计算器、日历等外部工具。
- 主要贡献：
  - 把“回答前判别”建模成一个可学习的 API 调用决策问题。
  - 用少量示例 + 自监督方式学习 route-to-tool，而不是手工写规则。
  - 证明 selective tool use 可以提高 factual QA 等任务的零样本表现。
- 为什么和你的题目有关：
  - 你的 `直接 RAG / 先改写 / 先追问 / 保守回答` 本质上也是一个 `pre-answer routing` 问题。
  - 这篇论文可以作为“回答前先决策”的早期 LLM 证据，用来支撑你论文中的路由器设计。

### 2. KG-ECO: Knowledge Graph Enhanced Entity Correction for Query Rewriting
- 作者：Jinglun Cai, Mingda Li, Ziyan Jiang, Eunah Cho, Zheng Chen, Yang Liu, Xing Fan, Chenlei Guo
- 首次公开：2023-02-21
- 链接：[arXiv:2302.10454](https://arxiv.org/abs/2302.10454)
- 摘要：
  - 论文关注对话系统中的 query rewriting，尤其是用户输入里存在实体错误时，系统如何先做实体纠错再继续后续回答。
  - 作者把问题拆成损坏实体片段检测、候选实体检索/重排，并引入知识图谱结构信息和实体文本描述。
- 主要贡献：
  - 提出面向 query rewriting 的实体纠错框架，而不是把错误输入直接送进响应模型。
  - 用 KG 结构邻接信息 + 文本描述联合提升实体纠错。
  - 在 few-shot 情况下尤其有效，这点对医疗长尾疾病名、药名、检查名很有借鉴价值。
- 为什么和你的题目有关：
  - 医疗问句常见错别字、简称、俗称、药品别名、部位名混淆；这类错误如果不在检索前修正，会直接把 RAG 带偏。
  - 这篇论文非常适合支撑你把 `rewrite` 细分成 `实体纠错 rewrite`。

### 3. Improving Intent Classification Using Unlabeled Data from Large Corpora
- 作者：Gabriel Bercaru, Ciprian-Octavian Truica, Costin-Gabriel Chiru, Traian Rebedea
- 首次公开：2023-02-03
- 链接：[MDPI / Mathematics 11(3):769](https://www.mdpi.com/2227-7390/11/3/769)
- 摘要：
  - 论文聚焦对话系统中的 intent classification，在标注数据稀缺时，如何从大规模无标注语料中高效挖掘相似文本做数据扩充。
  - 文章同时比较不同文本特征，并分析扩充数据在 few-shot 场景下是否真能带来 intent detection 改善。
- 主要贡献：
  - 提出从大规模无标注语料中自动挖掘相似样本的通用扩充流程。
  - 将 intent classification 的提升放到 few-shot 设置下系统评估。
  - 论文报告的结果显示：在较小标注规模下，扩充可带来稳定增益。
- 为什么和你的题目有关：
  - 你当前核心难点之一正是医学多意图识别数据少、人工标注贵、弱监督样本需要筛选。
  - 这篇论文与“从 5 万条患者问题里抽样、筛选、扩充训练集”的做法高度同构，可作为你的弱监督/近邻挖掘论证依据。

### 4. Almanac: Retrieval-Augmented Language Models for Clinical Medicine
- 作者：Cyril Zakka, Akash Chaurasia, Rohan Shad, Alex R. Dalal, Jennifer L. Kim, Michael Moor, Kevin Alexander, Euan Ashley, Jack Boyd, Kathleen Boyd, Karen Hirsch, Curt Langlotz, Joanna Nelson, William Hiesinger
- 首次公开：2023-03-01
- 链接：[arXiv:2303.01229](https://arxiv.org/abs/2303.01229)
- 摘要：
  - 论文提出 `Almanac`，把临床指南和治疗建议检索接入语言模型，目标是降低临床问答中的错误和风险性输出。
  - 作者在 130 个临床场景上，由 5 位临床医生评估 factuality、completeness、safety，报告 factuality 平均提升 18%。
- 主要贡献：
  - 这是可确认属于 `2023 Q1` 的早期医疗 RAG 代表作之一。
  - 论文把医疗问答中的 hallucination 问题直接转化为“先检索可信医学依据再回答”。
  - 使用医生评审而非只看自动指标，这对你后面设计医疗路由实验非常重要。
- 为什么和你的题目有关：
  - 虽然它没有重点讨论“先追问/先改写”，但它证明了在医疗场景里，回答前引入外部证据对 factuality 和 safety 非常关键。
  - 你的工作可以被表述为：在 `Almanac` 这类医疗 RAG 之前，再增加一个“输入质量控制层”。

### 5. Query2doc: Query Expansion with Large Language Models
- 作者：Liang Wang, Nan Yang, Furu Wei
- 首次公开：2023-03-14
- 链接：[arXiv:2303.07678](https://arxiv.org/abs/2303.07678)
- 摘要：
  - 论文提出 `query2doc`：先让 LLM 生成与查询相关的伪文档，再用伪文档对原始 query 做扩展，从而提升稀疏和稠密检索。
  - 作者强调伪文档里常含有可帮助 query disambiguation 的高相关信息，在无须微调的前提下提升 BM25 和 dense retrieval。
- 主要贡献：
  - 把 query rewriting / expansion 从短 query 改写成“伪文档扩展”，显著增强检索表达能力。
  - 给出一条很实用的思路：不是直接改用户问句，而是生成更适合检索器消费的中间表示。
  - 在 MS-MARCO、TREC DL 上报告稳定收益。
- 为什么和你的题目有关：
  - 你的 `rewrite` 不一定必须输出一个更漂亮的自然语言问句，也可以输出一个“检索友好型扩展表示”。
  - 对医疗患者输入而言，`症状 + 部位 + 持续时间 + 人群 + 既往史` 都可以被展开进伪文档，缓解原始提问过短、过口语、信息缺失的问题。

## 二、与《硕士论文进展》的关系

你的当前主线是：
- 识别患者问题中的多个医学意图。
- 判断输入是否清晰、医学相关、适合直接检索。
- 根据输入质量决定：直接 RAG、先改写、还是追问/保守回答。

这 5 篇论文和你当前设计的对应关系非常清楚：

| 你的模块 | 可对应论文 | 可借用点 |
| --- | --- | --- |
| 多意图识别 | Improving Intent Classification Using Unlabeled Data from Large Corpora | 无标注语料扩充、few-shot intent 分类、低成本数据扩张 |
| rewrite 前置层 | KG-ECO, Query2doc | 实体纠错 rewrite、检索友好扩展、query disambiguation |
| 回答前路由 | Toolformer | “先判断是否调用外部工具/检索/别的策略” 的路由思想 |
| 医疗 RAG 主干 | Almanac | 医疗 factuality/safety 评估，RAG 对幻觉治理的临床证据 |

如果把你的论文结构再压缩成一句话，可以写成：

> 在医疗 RAG 之前增加一个输入治理层，对患者问题执行多意图识别、输入质量评估与策略路由，决定是直接检索、先做 query rewrite、先追问澄清，还是触发保守回答，从而进一步降低医疗大模型幻觉。

从这个角度看：
- `Almanac` 说明“医疗 RAG 值得做”。
- `Toolformer` 说明“回答前策略选择值得单独做”。
- `KG-ECO` 和 `Query2doc` 说明“rewrite 不是装饰，而是检索质量核心变量”。
- `Intent Classification + Unlabeled Data` 说明“你的弱监督多意图识别路线是合理且有前例的”。

## 三、我可以直接学习的思路

### 1. 方法设计思路
- 把你的系统显式拆成 `ask / rewrite / retrieve / conservative answer` 四路，而不是只做一个连续分数。
- `rewrite` 再细分两类：
  - `entity correction rewrite`：修正文中疾病名、药名、检查名、部位名。
  - `retrieval-oriented expansion`：把患者口语输入扩成检索友好表示。
- `ask` 路由不一定只在“完全不懂”时触发，也可在“多意图冲突、关键槽位缺失、检索指向不唯一”时触发。
- `conservative` 路由可以单独定义为：不给诊断性结论，只给信息不足说明 + 建议补充项。

### 2. 实验方法
- 做两阶段实验，而不是把所有收益混在一起：
  - 第一阶段：只测输入治理层本身。
  - 第二阶段：把输入治理层接到医疗 RAG 上，测最终问答质量。
- 对 `rewrite` 做分桶消融：
  - 无 rewrite
  - 只做实体纠错
  - 只做检索扩展
  - 两者都做
- 对 `ask` 做触发消融：
  - 从不追问
  - 低阈值追问
  - 高阈值追问
- 对 `route` 做 oracle 上限实验：
  - 用人工标注的最优路由当上限，估计路由模块还有多少提升空间。

### 3. 可用数据集思路
- 继续以你现有 `MedDialog-CN / 2010.txt` 患者问题为主。
- 从中额外构造三类“低质量输入”测试集：
  - 多意图混合：一个问题里同时问症状、用药、检查、挂号。
  - 信息缺失：缺年龄、部位、持续时间、诱因、既往史。
  - 实体噪声：错别字、俗称、简称、同音、非标准药名。
- `rewrite` 模块可人工标一个小型 gold 集：
  - 原始问句
  - 应纠正的实体
  - 应补全的检索表达
  - 是否应先追问

### 4. 评测指标
- 多意图识别：
  - `micro-F1`
  - `macro-F1`
  - `hamming loss`
- 路由：
  - route accuracy
  - macro-F1
  - confusion matrix
  - 高风险类召回率
- rewrite：
  - 检索前后 `Recall@k`, `MRR`, `nDCG`
  - top1 是否命中 gold evidence
  - 实体纠错准确率
- ask：
  - 是否成功减少错误回答
  - 追问后最终检索命中率
  - 追问成本：平均追问轮次
- 最终医疗 QA：
  - factuality
  - safety
  - completeness
  - physician preference / expert pairwise judgment

## 四、可直接引用到论文里的点

### 1. 论文动机可引用点
- 医疗大模型的风险不只来自“回答阶段 hallucination”，也来自“输入阶段欠清晰、欠完整、实体错误、多意图混杂”。
- 因而在医疗 RAG 前加入输入治理层，是对现有医疗 RAG 框架的自然补强。

### 2. 方法论可引用点
- `Toolformer` 可支撑“回答前先判别是否需要外部工具/检索”的合理性。
- `KG-ECO` 可支撑“query rewriting 应包括实体纠错，而不只是语义改写”。
- `Query2doc` 可支撑“rewrite 的目标是提升检索，不必拘泥于保持原句形式”。
- `Almanac` 可支撑“医疗 RAG 对 factuality 与 safety 有直接帮助，但仍需要精细的输入控制层”。
- `Intent Classification Using Unlabeled Data` 可支撑“少标注 + 大规模无标注患者语料”的弱监督扩充路线。

### 3. 你这条线最值得强调的新意
- 不是单纯做医疗问答。
- 不是单纯做医疗 RAG。
- 不是单纯做 intent classification。
- 而是把三者串起来，形成一个面向医疗 hallucination 的 `输入理解 -> 策略路由 -> 检索增强回答` 框架。

## 五、这次时间桶的结论

- `2023 Q1` 这个桶里，直接命中“医疗 RAG + ask/rewrite/conservative 路由”三者同时具备的论文很少。
- 但已经能看见三条后来会汇合的技术线：
  - 医疗 RAG：`Almanac`
  - 回答前路由：`Toolformer`
  - 检索前 rewrite / disambiguation：`KG-ECO` 与 `Query2doc`
- 你的硕士论文正好处在这三条线的交叉点上，差异化空间是存在的。
