## 论文 1

### CLAM: Selective Clarification for Ambiguous Questions with Generative Language Models

- 作者：Lorenz Kuhn, Yarin Gal, Sebastian Farquhar
- 首次公开时间：2022-12-15（arXiv v1）
- 链接：[arXiv](https://arxiv.org/abs/2212.07769)

#### 摘要

论文直接研究一个和 hallucination 高度相关的问题：面对**歧义问题**，语言模型往往不追问，而是直接假设一种解释并给出可能错误的答案。作者提出 `CLAM`，让模型先判断问题是否含糊、是否需要澄清；如果需要，就先生成澄清问题，再在获得补充信息后回答。

#### 主要贡献

1. 把“是否需要先问再答”建模成显式流程，而不是默认所有问题都可直接回答。
2. 让模型同时完成三个子任务：**歧义检测**、**澄清问题生成**、**澄清后的最终回答**。
3. 设计了带“特权信息”的用户模拟评测方法，使多轮 clarification 能自动化评估。
4. 证明 selective clarification 比直接回答更稳，尤其适合混合了清晰问题和歧义问题的真实场景。

#### 和你论文的关系

这篇论文和你的 `ask / direct / conservative` 路由最接近。你当前想解决的是：患者输入是否足够清楚，能否安全直接进入医疗 RAG。`CLAM` 给你的直接启发是：

- `Q_route` 不应只是经验规则，而可以被形式化为**先判别，再决定是否追问**。
- 医疗问题里缺少关键槽位时，直接生成医学建议很容易导致“带假设回答”，这本质上就是输入侧 hallucination。
- 你的系统可以把 `need_ask` 单独建成分类目标，而不是把追问当作文案模板问题。

#### 你可以学习的思路、实验方法、数据集、实验设计、评测指标、可引用点

- 思路：把“不该直接答”的判别前移到生成前。
- 实验方法：比较 `always-answer`、`ask-when-unclear`、`conservative-abstain` 三种策略。
- 数据集：从你的患者问题中人工构造一批“缺部位 / 缺持续时间 / 缺对象 / 缺检查结果”的歧义样本。
- 实验设计：同时测 `need_ask` 识别准确率和追问后下游检索/回答改善。
- 评测指标：`route accuracy`、`ask recall`、`unsafe direct answer rate`、`post-clarification gain`。
- 可引用点：可靠系统应先识别歧义并主动澄清，而不是默认用户输入已经足够明确。

## 论文 2

### Towards Multi-label Unknown Intent Detection

- 作者：Yawen Ouyang, Zhen Wu, Xinyu Dai, Shujian Huang, Jiajun Chen
- 首次发表/可检索时间：2022-10（COLING 2022，10 月 12 日到 17 日）
- 链接：[ACL Anthology](https://aclanthology.org/2022.coling-1.52/)

#### 摘要

这篇论文指出，传统 unknown intent detection 往往假设一句话只有一个意图，但真实输入经常同时带多个意图，甚至同时包含**已知意图和未知意图**。作者提出更符合真实场景的任务 `multi-label unknown intent detection`，并给出 `AIK` 方法来判断一个输入中是否存在未知意图。

#### 主要贡献

1. 把 unknown intent detection 从单意图场景推广到**多意图、多标签**场景。
2. 明确定义 `mixed OOD utterance`：一句话中同时含有已知意图和未知意图。
3. 提出 `AIK`，核心思想是先预测意图数量，再判断这些意图是否都能被已知意图集合覆盖。
4. 在 MixSNIPS 和 MultiWOZ 2.3 上验证方法有效，尤其改善 mixed OOD 样本识别。

#### 和你论文的关系

这篇论文非常适合支撑你现在的**多意图识别 + 保守路由**。你的患者输入里常见两种难点：

- 一句话同时问多个医学问题。
- 一部分内容属于可处理医学意图，另一部分可能偏生活化、模糊化、跨域或暂时不可处理。

这和 `mixed OOD` 的设定高度相似。对你来说，最重要的不是把所有内容都硬判到 12 类里，而是识别：

- 这条输入是否存在**未知/越界/不安全意图**。
- 是否需要触发 `conservative` 或 `ask` 路由。

#### 你可以学习的思路、实验方法、数据集、实验设计、评测指标、可引用点

- 思路：在多标签意图分类之前或之后，加一个 `unknown / mixed OOD` 检测层。
- 实验方法：比较 `纯多标签分类` 与 `多标签分类 + unknown intent gate`。
- 数据集：在你现有 12 类标签外，额外构造 `非医学`、`信息不足`、`跨域诉求`、`无法安全判断` 等开放集样本。
- 实验设计：把 test 集拆成 `纯已知意图`、`纯未知意图`、`已知+未知混合意图` 三类。
- 评测指标：`AUROC`、`FPR95`、`mixed OOD recall`、`unsafe route rate`。
- 可引用点：真实用户输入常同时携带多个已知/未知意图，系统需要先识别开放集风险，再决定是否进入下游处理。

## 论文 3

### Explicit Query Rewriting for Conversational Dense Retrieval

- 作者：Hongjin Qian, Zhicheng Dou
- 首次发表/可检索时间：2022-12（EMNLP 2022）
- 链接：[ACL Anthology](https://aclanthology.org/2022.emnlp-main.311/)

#### 摘要

论文关注对话检索中的上下文依赖问题。作者指出，查询往往因为指代、省略而不自洽，已有方法要么只做 query rewriting，要么只做 context modeling。论文提出 `CRDR`，把两者统一起来，只对原始问题做必要修改，从而得到更高效、更准确的可检索查询。

#### 主要贡献

1. 把 `query rewriting` 和 `context modeling` 放进一个统一框架。
2. 不是从零生成新问题，而是只做**必要修改**，降低改写漂移风险。
3. 用改写监督信号反过来增强上下文建模。
4. 在 TREC CAsT-19 / CAsT-20 上同时提升 query rewriting 质量和 context-aware ranking 效果。

#### 和你论文的关系

这篇论文对你的 `rewrite` 路由特别关键。你当前场景里很多患者输入不是缺信息，而是：

- 指代不清。
- 省略主语、对象、时间。
- 口语噪声多，但核心需求其实足够。

这类输入不一定要追问，更适合先做 `denoise / rewrite` 再进检索。`CRDR` 给你的方法论是：**改写应尽量少改，只补齐检索真正需要的缺口**。这能降低医疗场景中“改写把用户意图改歪”的风险。

#### 你可以学习的思路、实验方法、数据集、实验设计、评测指标、可引用点

- 思路：把 `rewrite` 定义为“必要修改”而不是“完全重写”。
- 实验方法：比较 `raw query`、`full rewrite`、`minimal rewrite` 三种输入形式。
- 数据集：从 MedDialog-CN 中抽出含指代、省略、口语残缺的问题，人工写 `minimal rewrite`。
- 实验设计：同时评估改写质量与下游检索质量，避免只看文本相似度。
- 评测指标：`Recall@k`、`MRR`、`rewrite faithfulness`、`intent preservation rate`。
- 可引用点：检索前改写的目标不是让句子更漂亮，而是以最小改动恢复可检索、意图一致的独立查询。

## 论文 4

### PACIFIC: Towards Proactive Conversational Question Answering over Tabular and Textual Data in Finance

- 作者：Yang Deng, Wenqiang Lei, Wenxuan Zhang, Wai Lam, Tat-Seng Chua
- 首次发表/可检索时间：2022-12（EMNLP 2022）
- 链接：[ACL Anthology](https://aclanthology.org/2022.emnlp-main.469/)

#### 摘要

论文提出 `PACIFIC` 数据集和 `PCQA` 任务，核心是把**澄清问题生成**和**问答**统一进一个主动式流程。场景在金融领域，但研究问题本质上是：当上下文混合表格和文本、用户问题不够直接时，系统如何主动追问，再完成后续回答。

#### 主要贡献

1. 提出 `Proactive Conversational Question Answering (PCQA)`，把追问和回答放在同一任务里。
2. 发布 `PACIFIC` 数据集，强调 proactivity、numerical reasoning、hybrid context。
3. 给出 `UniPCQA`，用统一 Seq2Seq 方式处理多个子任务。
4. 明确评估 clarification generation 与 CQA 的联动，而不是把追问单独孤立评估。

#### 和你论文的关系

虽然不是医疗数据，但它对你有两个很直接的启发：

- `ask` 不是外挂模块，而应被纳入整个 pipeline 评估。
- 追问的价值应体现在**澄清后能否提升最终回答/检索质量**，而不是只看追问句子本身。

你完全可以把这个思想迁移到医疗 RAG：患者问题如果缺症状部位、持续时间、检查结果、年龄/孕产信息，系统先追问，再决定是否检索与回答。

#### 你可以学习的思路、实验方法、数据集、实验设计、评测指标、可引用点

- 思路：将 `clarification + answering` 视作一个联合任务。
- 实验方法：测 `追问质量` 和 `追问后回答质量` 的联动增益。
- 数据集：在你的人工评测集中为每条 `need_ask` 样本补一条 gold clarifying question 和 gold missing slots。
- 实验设计：设定 `追问前` 与 `追问后` 两阶段评测。
- 评测指标：`clarification utility`、`slot completion rate`、`post-ask retrieval gain`、`final answer safety`。
- 可引用点：主动式问答不应把澄清问题视为附属模块，而应把它作为提升最终正确性与安全性的核心步骤。

## 论文 5

### CONQRR: Conversational Query Rewriting for Retrieval with Reinforcement Learning

- 作者：Zeqiu Wu, Yi Luan, Hannah Rashkin, David Reitter, Hannaneh Hajishirzi, Mari Ostendorf, Gaurav Singh Tomar
- 首次发表/可检索时间：2022-12（EMNLP 2022）
- 链接：[ACL Anthology](https://aclanthology.org/2022.emnlp-main.679/)

#### 摘要

论文研究如何把对话问题改写成独立可检索的问题，并且不要求重训检索器。作者提出 `CONQRR`，用 reinforcement learning 直接朝向检索效果优化 query rewriting，而不是只学习和人工 rewrite 文本接近。

#### 主要贡献

1. 把 query rewriting 的目标从“像人工改写”转成“提升下游检索”。
2. 用 reinforcement learning 直接以 retrieval reward 训练改写模型。
3. 兼容 off-the-shelf retriever，不需要改动既有检索引擎。
4. 对 out-of-domain dialogue 和零改写监督场景都表现出较强鲁棒性。

#### 和你论文的关系

这篇论文和你未来的医疗 RAG 系统结合度很高，因为它强调：**改写是否有效，应由检索结果来裁判**。这对你的 `rewrite` 路由尤其重要：

- 医疗改写不能只追求通顺。
- 更关键的是改写后是否检索到更相关、更安全的医学证据。
- 你的 `Q_route` 以后可以不只判断“要不要 rewrite”，还可以判断“rewrite 后是否真的提升 retrieval”。

#### 你可以学习的思路、实验方法、数据集、实验设计、评测指标、可引用点

- 思路：把改写质量和检索结果直接绑定。
- 实验方法：在你的实验里加入 `rewrite-for-text` 与 `rewrite-for-retrieval` 两种目标的对比。
- 数据集：用现有患者问题，人工标部分 rewrite，并结合检索回来的医学证据做弱监督。
- 实验设计：比较 `直接检索`、`规则改写后检索`、`学习式改写后检索`。
- 评测指标：`Recall@k`、`nDCG@k`、`evidence relevance`、`rewrite helpfulness`。
- 可引用点：面向检索增强系统，query rewriting 的真实价值应由下游检索收益而非表面文本相似度决定。

## 与硕士论文进展的关系

结合 `[[硕士论文进展]]`，今天这一桶最值得你吸收的是一个更清晰的**输入治理四层框架**：

1. **多意图与未知意图识别**：对应 `Towards Multi-label Unknown Intent Detection`。
2. **是否需要先问再答**：对应 `CLAM`。
3. **如果不必追问，是否需要 rewrite / denoise**：对应 `Explicit Query Rewriting` 与 `CONQRR`。
4. **追问和最终回答要联合评估**：对应 `PACIFIC`。

这和你现在的 `Q_sem` 可以形成更强的解释：

```text
患者输入
-> 多意图识别 / unknown intent detection
-> 信息充分性与安全性判别
-> 路由决策：direct RAG / rewrite / ask / conservative
-> 检索
-> 生成
```

你的创新点可以因此更明确地表述为：

- **不是直接优化医疗回答生成，而是先治理输入是否适合进入 RAG。**
- **不是把 hallucination 只理解为生成阶段造假，而是把“带缺失假设直接回答”也视为输入侧 hallucination 风险。**
- **不是单纯分类意图，而是把意图识别、澄清触发、改写触发、保守路由联成一个前置控制器。**

## 我可以直接学习和复用的内容

### 1. 可提炼成论文方法的思路

- 在多标签意图分类之外，增加 `unknown / mixed intent` 检测。
- 把 `ask / rewrite / conservative / direct` 建成显式四路路由任务。
- 把 `rewrite` 分成两类：`minimal rewrite` 和 `denoise rewrite`。
- 把“是否该回答”做成前置判别模块，而不是生成后补救。

### 2. 可直接借鉴的实验方法

- 四路系统对比：`direct RAG`、`rewrite-then-RAG`、`ask-then-RAG`、`conservative answer`。
- 加一个 `route oracle` 上界实验：如果路由全对，最终上限能提高多少。
- 做 `已知单意图`、`已知多意图`、`已知+未知混合意图`、`信息不足` 四类分层评测。
- 把 route accuracy 和 downstream retrieval / answer safety 一起汇报。

### 3. 可构造的数据集

- 在现有 5 万患者问题中继续筛：
  - 多意图样本
  - 未知/越界意图样本
  - 缺关键槽位样本
  - 适合改写但不必追问样本
- 新增一套人工标注字段：
  - `intent labels`
  - `need_ask`
  - `need_rewrite`
  - `need_conservative`
  - `unknown_or_mixed_intent`
  - `missing_slots`

### 4. 可用的实验设计

- `route first` vs `always retrieve first`
- `minimal rewrite` vs `full rewrite`
- `ask` 触发前后检索效果对比
- `unknown intent gate` 加与不加的安全性对比

### 5. 建议增加的评测指标

- 多标签分类：`micro-F1`、`macro-F1`、`hamming loss`
- 路由：`route accuracy`、`ask recall`、`rewrite precision`
- 安全性：`unsafe direct answer rate`
- 检索：`Recall@k`、`MRR`、`nDCG@k`
- 追问价值：`slot completion rate`、`post-clarification gain`
- 开放集：`AUROC`、`FPR95`、`mixed OOD recall`

## 这次最值得写进论文的可引用点

1. **真实医疗输入经常不是“单一、完整、可直接检索”的标准问题，而是多意图、缺槽位、夹杂未知诉求的混合输入。**
2. **可靠系统应先判断能否安全直接处理，再决定 ask / rewrite / conservative，而不是默认所有问题都可回答。**
3. **query rewriting 的价值不在于文本更自然，而在于是否提升下游检索并保持原始意图。**
4. **澄清问题不应单独评测，必须看其是否真正减少检索歧义、补齐关键医学槽位并提升最终回答安全性。**
5. **输入治理可以被看作医疗 RAG 的前置控制器，这比单独讨论生成阶段 hallucination 更容易形成你的论文创新点。**
