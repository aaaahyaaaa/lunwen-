## 论文 1

### The StatCan Dialogue Dataset: Retrieving Data Tables through Conversations with Genuine Intents

- 作者：Kostadin Kushlev, Ryan Kuz, Danial Khosravi, Sandeep Guru, Jimmy Lin
- 首次公开时间：2023-04-03（arXiv v1）
- 链接：[arXiv](https://arxiv.org/abs/2304.01412)

#### 摘要

论文构建了一个面向真实用户意图的对话式表格检索数据集，强调用户查询往往不是单轮、完整、规范的，而是带有上下文依赖、补充约束和真实意图跳转。作者把重点放在 **genuine intents** 下的 conversational retrieval，而不是理想化、单句式检索输入。

#### 主要贡献

1. 提供更接近真实用户行为的 conversational retrieval 数据资源。
2. 突出展示多轮检索中意图表达不完整、上下文依赖强的问题。
3. 为后续 query understanding、query rewrite、route-before-retrieve 提供了评测土壤。

#### 和你论文的关系

这篇不是医疗论文，但它给你的一个很重要提醒是：**真实输入往往先天不适合直接检索**。你的患者问题同样有口语、省略、跨意图和补充约束，这意味着你现在的 `Q_sem` 不是附加模块，而应被当成 RAG 前的必要入口。

#### 你可以学习的思路、实验方法、数据集、实验设计、评测指标、可引用点

- 思路：把“真实输入不规范”本身当作系统设计前提，而不是噪声例外。
- 实验方法：从你现有患者问题中挑出省略、追问式、跨轮依赖样本，单独构造 `context-dependent subset`。
- 数据集：仿照其 genuine intent 取样方式，补一个“真实患者问题困难子集”。
- 实验设计：比较 `原始问题直接检索` 与 `前置理解后再检索`。
- 评测指标：`Recall@k`、`MRR`、`route accuracy`、`unsafe direct answer rate`。
- 可引用点：真实场景中的查询往往带上下文依赖和不完整意图，因此对输入进行前置理解是必要而不是可选的。

## 论文 2

### Effective Open Intent Classification with K-center Contrastive Learning and Adjustable Decision Boundary

- 作者：Xiaokang Liu, Jianquan Li, Jingjing Mu, Min Yang, Ruifeng Xu, Benyou Wang
- 首次公开时间：2023-04-20（arXiv v1）
- 链接：[arXiv](https://arxiv.org/abs/2304.10220)

#### 摘要

论文研究 open intent classification：系统既要把已知输入分到已有意图类别，也要识别新的未知意图。作者提出 K-center contrastive learning 和 adjustable decision boundary learning，通过增强已知/未知边界学习，提高开放集场景下的分类和拒识效果。

#### 主要贡献

1. 明确把意图识别从封闭集分类扩展到 **open intent classification**。
2. 用对比学习和可调决策边界改善已知/未知意图的区分。
3. 让系统在保留已知意图分类能力的同时，更稳地拒识未知输入。

#### 和你论文的关系

这篇和你的 `conservative` 路由直接相关。你的系统现在做的是 12 类多标签分类，但医疗 RAG 真正危险的情况之一，是用户问题 **根本不在已知标签空间内**，或者混有非医学、越界、难以负责回答的内容。它支持你把“未知/超域判别”从附属规则升级成独立模块。

#### 你可以学习的思路、实验方法、数据集、实验设计、评测指标、可引用点

- 思路：在多标签意图分类器之外显式加入 `unknown intent gate`。
- 实验方法：比较 `仅多标签分类` 与 `多标签分类 + 开放集拒识`。
- 数据集：补充非医学、营销、法律、价格咨询、模糊抱怨等开放集样本。
- 实验设计：分别测试 `已知`、`未知`、`已知+未知混合` 三类输入。
- 评测指标：`AUROC`、`FPR95`、`OOD recall`、`unsafe route rate`。
- 可引用点：面向真实用户输入的系统需要识别未知意图，而不是把所有输入强制映射到既有标签空间。

## 论文 3

### ViMQ: A Dataset of Visual Medical Queries for Image-Based Diagnosis Assistance

- 作者：Ramkrishna Gupta, Shreyasi Pathak, Dibyajyoti Saha, Sundeep Teki, Animesh Mukherjee, Tirthankar Ghosal
- 首次公开时间：2023-04-27（arXiv v1）
- 链接：[arXiv](https://arxiv.org/abs/2304.14405)

#### 摘要

论文提出一个面向医疗问诊辅助的视觉医疗查询数据集，关注现实场景中患者通过图片和自然语言共同表达求助意图的情况。它的重点不是 hallucination 治理本身，而是展示 **医疗查询理解并不总是标准文本分类问题**，真实输入可能是多模态、信息缺失且表达不标准的。

#### 主要贡献

1. 给出更贴近现实医疗咨询场景的视觉+文本查询资源。
2. 强调患者查询表达天然存在不完整、弱规范、多模态依赖。
3. 为医疗查询理解和诊断辅助提供更真实的输入形式基准。

#### 和你论文的关系

这篇与 `medical RAG` 的关系是“输入形态层补充”，不是“路由算法层直接支撑”。但它很适合放在文献综述里说明：**医疗输入质量问题不只来自语言歧义，也来自模态不完整、实体缺失和表达弱规范**。这能帮你论证 `Q_text / Q_med / Q_route` 的必要性。

#### 你可以学习的思路、实验方法、数据集、实验设计、评测指标、可引用点

- 思路：把医疗输入理解视作更广义的 query quality problem，而不只是文本意图分类。
- 实验方法：在你的数据讨论部分加入“现实患者输入可能弱规范、缺槽位、带图片”的外部动机。
- 数据集：即便你不做多模态，也可以借它论证后续系统扩展方向。
- 实验设计：当前可先只做文本，但在讨论中保留“多模态输入下 ask/rewrite 更重要”的扩展点。
- 评测指标：可参考其任务拆分方式讨论未来扩展评价。
- 可引用点：医疗求助输入常常弱规范且信息不完整，因此在检索和回答前进行输入质量控制具有现实必要性。

## 论文 4

### Out-of-Domain Intent Detection Considering Multi-Turn Dialogue Contexts

- 作者：Hao Lang, Yinhe Zheng, Binyuan Hui, Fei Huang, Yongbin Li
- 首次公开时间：2023-05-05（arXiv v1）
- 链接：[arXiv](https://arxiv.org/abs/2305.03237)

#### 摘要

论文关注多轮对话中的 OOD intent detection。作者指出，已有 OOD 研究多停留在单轮输入，但真实系统里用户意图常依赖前文。为此提出 context-aware OOD intent detection 框架，从多轮上下文中抽取更稳健的表示，用来识别超域输入。

#### 主要贡献

1. 把 OOD intent detection 从单轮扩展到 **multi-turn context**。
2. 用上下文建模提高超域判别的鲁棒性。
3. 更接近真实对话系统部署场景。

#### 和你论文的关系

这篇对你非常有价值，因为医疗咨询并不总是独立单句。即便你当前主要处理单句患者问题，也可以借它说明：**一旦系统接入多轮追问，OOD 判别必须结合上下文，而不是只看当前句子**。这正好补你的 `ask` 路由后续延展逻辑。

#### 你可以学习的思路、实验方法、数据集、实验设计、评测指标、可引用点

- 思路：把 `unknown / non-medical / risky` 判别做成可接多轮上下文的 gate。
- 实验方法：在当前单轮数据上先做基本 OOD，后续再补小规模多轮样本。
- 数据集：从 MedDialog 原始对话中抽取前后文，构造 `上下文改变当前意图` 的子集。
- 实验设计：比较 `仅当前句` 和 `当前句+前文` 的 OOD 检测效果。
- 评测指标：`AUROC`、`AUPR`、`FPR95`、`context-sensitive route accuracy`。
- 可引用点：在真实对话场景中，超域意图检测需要结合多轮上下文，不能假设输入是孤立单句。

## 论文 5

### ConQueR: Contextualized Query Reduction using Search Logs

- 作者：Hye-young Kim, Minjin Choi, Sunkyung Lee, Eunseong Choi, Young-In Song, Jongwuk Lee
- 首次公开时间：2023-05-22（arXiv v1）
- 链接：[arXiv](https://arxiv.org/abs/2305.12662)

#### 摘要

论文提出 conversational search 中的 query reduction 问题：不是无限制重写，而是在结合上下文后，**删去冗余、保留必要约束、还原最适合检索的查询表达**。作者利用 search logs 训练 contextualized query reduction 模型，使其更接近真实搜索行为。

#### 主要贡献

1. 把 conversational rewrite 细化为 **query reduction**，强调“减法式”改写。
2. 利用真实 search logs 学习更贴近用户行为的 reformulation。
3. 为 conversational retrieval 提供更稳的检索前输入形式。

#### 和你论文的关系

这篇很适合支撑你的 `rewrite` 路由细化。医疗输入改写不一定总是补充信息，也可能是 **删掉无关铺垫、保留症状和约束条件**。这对减少误检索很重要，因为很多患者问题前半句是背景叙述，真正有检索价值的信息只占一小部分。

#### 你可以学习的思路、实验方法、数据集、实验设计、评测指标、可引用点

- 思路：把 rewrite 再拆成 `reduction` 与 `completion` 两类，不要只做统一重写。
- 实验方法：比较 `raw query`、`reduction rewrite`、`completion rewrite` 三种进入检索的效果。
- 数据集：从患者问题中标注“冗余背景片段”和“核心检索片段”。
- 实验设计：同时测 rewrite 的意图保真和检索增益。
- 评测指标：`intent preservation rate`、`Recall@k`、`MRR`、`rewrite helpfulness`。
- 可引用点：高质量检索前处理不一定是生成更长查询，也可以是保留关键信息、去掉冗余信息的 reduction。

## 论文 6

### ConvGQR: Generative Query Reformulation for Conversational Search

- 作者：Fengran Mo, Kelong Mao, Yutao Zhu, Yihong Wu, Kaiyu Huang, Jian-Yun Nie
- 首次公开时间：2023-05-25（arXiv v1）
- 链接：[arXiv](https://arxiv.org/abs/2305.15645)

#### 摘要

论文提出生成式 conversational query reformulation 方法，把对话上下文转为更适合检索器的表达，并显式优化下游检索效果。它强调 reformulation 不只是生成更通顺的句子，而是要生成 **对检索真正有帮助的查询**。

#### 主要贡献

1. 把 generative reformulation 和下游 retrieval performance 直接对齐。
2. 让 conversational rewrite 从表面语言改写转向检索目标驱动。
3. 提供更强的 conversational retrieval reformulation baseline。

#### 和你论文的关系

这篇是你 `rewrite` 路由最直接的技术支撑之一。你的硕士论文里如果要证明“先改写再检索”有价值，关键不是看改写句是否更通顺，而是看 **证据召回是否提升、错误证据是否减少、最终回答是否更安全**。这篇给了你一个很明确的评价方向。

#### 你可以学习的思路、实验方法、数据集、实验设计、评测指标、可引用点

- 思路：把 rewrite 的优化目标从“像人类重述”转成“帮助检索”。
- 实验方法：比较 `直接检索`、`规则改写`、`生成式改写`。
- 数据集：从你现有 5 万条患者问题中抽一个 `需要改写` 子集做专门实验。
- 实验设计：先人工标 `need_rewrite`，再测改写前后检索与回答质量差异。
- 评测指标：`Recall@k`、`nDCG`、`evidence relevance`、`unsafe answer rate`。
- 可引用点：查询改写的价值应通过下游检索与回答质量来衡量，而不是只看语言表面质量。

## 论文 7

### Query Understanding in the Age of Large Language Models

- 作者：Wei Liu, Xiaoyan Yang, Yiqun Liu, Sujian Li, Yunqiu Xu, Heyan Huang, Zhifang Sui, Dacheng Tao
- 首次公开时间：2023-06-28（arXiv v1）
- 链接：[arXiv](https://arxiv.org/abs/2306.16004)

#### 摘要

论文从更高层面讨论 LLM 时代的 query understanding，提出 query understanding 可以围绕四类核心能力展开：`discriminate`、`infer`、`edit`、`expand`。作者强调大模型不应只被看成回答器，也可以被用作查询解释器、缺失信息补全器、改写器和意图推断器。

#### 主要贡献

1. 重新组织 query understanding 在 LLM 时代的任务版图。
2. 明确区分判别、推断、编辑、扩展几类前置处理能力。
3. 为“回答前先理解查询”提供系统性表述框架。

#### 和你论文的关系

这篇与您的总体框架高度一致，几乎可以拿来做文献综述里的总括性支撑。你当前的 `Q_sem + ask/rewrite/direct/conservative` 可以直接对应到：

- `discriminate`：判断是否清晰、是否医学相关、是否需要拒识。
- `infer`：推断隐含意图、多意图、缺失槽位。
- `edit`：做 query rewrite / reduction / correction。
- `expand`：为检索补全必要上下文。

#### 你可以学习的思路、实验方法、数据集、实验设计、评测指标、可引用点

- 思路：把你的系统明确写成一个 `query understanding front-end for medical RAG`。
- 实验方法：按 `discriminate / infer / edit / expand` 重组模块描述与消融实验。
- 数据集：现有患者问题可分别映射到 `ask`、`rewrite`、`direct`、`conservative` 四类。
- 实验设计：做模块级消融，而不是只报告最终 route 结果。
- 评测指标：除分类 F1 外，增加 `route-level success`、`post-route retrieval gain`、`unsafe answer reduction`。
- 可引用点：在 LLM 时代，查询理解不应局限于意图分类，而应覆盖判别、推断、编辑和扩展等回答前处理能力。
